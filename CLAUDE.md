# CLAUDE.md

## 프로젝트 개요

한국어 사전 데이터를 기반으로 글자/음절 빈도 및 N-gram을 분석하는 C 언어 프로젝트.

## 작업 원칙

### 불명확한 요청 처리
- 요청의 의도가 명확하지 않으면 **추측하여 진행하지 않는다**
- 반드시 다음 순서를 따른다:
  1. 이해한 내용과 가정을 명시적으로 정리한다
  2. 실행 계획(plan)을 제시한다
  3. 사용자에게 확인을 받은 후 진행한다
- 과거 실수 사례: `의미만 남겨라`는 요청을 `-`, `^` 포함 단어를 "제거"로 해석했으나, 실제 의도는 구분자를 제거하고 단어를 합쳐 **모든 레코드를 유지**하는 것이었음

## 작업 규칙

### 파일 네이밍
- 소스: `step{N}/step0{N}_{작업명}.c`
- 결과: `step{N}/{작업명}_result.txt`
- 실행 파일: `step{N}/step0{N}_{작업명}` (확장자 없음)

### 토큰 절약
- **작업 전**: `feedback/` 폴더의 관련 기록을 먼저 확인 — 파일을 직접 읽기 전에 반드시 확인
- **작업 후**: 주요 작업(데이터 전처리, 새 step 추가 등)은 `feedback/`에 기록

### 절대 직접 읽지 않는 파일 (Read 도구 사용 금지)

| 파일 | 크기 | 알려진 사실 |
|------|------|------------|
| `data/korean-dict/kr_korean.csv` | 9.7MB / 508,142줄 | 원본 사전, 형식: `낱말,품사`, UTF-8 BOM |
| `data/korean-dict/kr_korean_simple.csv` | 5.5MB / 508,142줄 | 전처리 완료, 낱말만, 구분자(-,^,공백) 제거됨 |
| `step1/charfreq_result.txt` | 2,467줄 | 형식: `글자\t빈도`, 빈도 내림차순, 상위: 다(90519) 하(58848) 기(28632) |
| `step3/bigram_stats.txt` | 172,685줄 | 형식: `^가`/`가$`/`가나`\t빈도 |
| `step4/trigram_stats.txt` | 659,449줄 | 형식: `^^가`/`^가나`/`가나$`/`가나다`\t빈도 |

내용 확인이 필요할 때는 `head -N`, `grep`, `wc -l` 만 사용한다.

### 새 step 작성 전 체크리스트

이전 step 코드를 확장할 때 미리 확인 (feedback/heavy_io_analysis.md 참고):

1. **키 타입**: n이 커질수록 키 범위 초과 → `uint64_t` 필요 여부 확인
2. **스택 배열**: 100KB 초과 배열은 반드시 `malloc` 힙 할당
3. **파일 형식**: START 토큰 수 = n-1개, 파서 수정 범위 확정 후 코딩
4. **필드명**: 이전 step 코드 복붙 후 구조체 필드명 불일치 확인

### n-gram 경계 토큰 규칙

- n-gram에서 START 토큰은 **n-1개** 붙인다
  - 2-gram: `^가나` (START 1개)
  - 3-gram: `^^가나` (START 2개)
  - 4-gram: `^^^가나` (START 3개)

### 컴파일
```bash
gcc -O2 -o step{N}/실행파일명 step{N}/소스파일.c
```

### 코드 스타일
- 함수마다 한국어 주석 작성
- 전역 상수는 `#define`으로 상단에 정의
- 결과 출력 형식: `항목\t빈도` (탭 구분, 빈도 내림차순)
- 결과는 파일 스트림(`FILE*`)으로 저장, 경로는 인자로 받아 기본값 제공

### step 간 의존 관계
- 각 step은 이전 step의 결과 파일을 입력으로 받는다
- step2는 step1의 `charfreq_result.txt`를 입력으로 사용

## 현재 진행 단계

| Step | 내용 | 상태 |
|------|------|------|
| step1 | 완성형 글자(가-힣) 빈도 집계 | 완료 |
| step2 | 유니그램 확률 분포 기반 샘플링 (10세트 × 10글자) | 완료 |
| step3 | 바이그램 빈도 집계 + 시작/끝 토큰 + 샘플링 + 사전 검증 | 완료 |
| step4 | 트라이그램 빈도 집계 + 샘플링 + 사전 검증 | 완료 |
| report | 바이그램 vs 트라이그램 비교 리포트 (Python) | 완료 |
